Bocetos de la aplicación

En esta página presentamos bocetos de baja fidelidad (wireframes) para las pantallas intermedias que verán los distintos usuarios de Bistro FDI (cliente, camarero, cocinero y gerente). Incluimos una explicación textual de cada pantalla y cómo se navega entre ellas, paso a paso, desde la página principal.

Nota de diseño: para la creación de pedidos, el cliente usará principalmente un móvil en vertical; camareros y cocineros utilizarán tablets con interfaces sencillas; y el gerente usará un ordenador con vistas pensadas para gestión y formularios.

Boceto 1 — Inicio (Desktop)

Qué se ve: imagen principal, descripción breve del proyecto y accesos rápidos a las acciones más frecuentes.

Navegación:

  • “Iniciar sesión / Registrarse” lleva al Boceto 2.
  • “Ver carta” lleva al Boceto 3.
  • “Mis pedidos / Perfil” lleva al Boceto 8 (si el usuario está identificado).

Boceto 2 — Login / Registro

Qué se ve: formulario de login y registro (cliente) con datos básicos y avatar.

Navegación:

  • Tras login correcto, se vuelve a Inicio (Boceto 1) o se redirige a la Carta (Boceto 3).
  • Tras registro, el usuario queda creado como “cliente” y puede comenzar el flujo de pedido.

Boceto 3 — Carta: categorías y productos (Móvil)

Qué se ve: selector del tipo de pedido (Local / Llevar), categorías y listado de productos con botones rápidos para añadir.

Navegación:

  • Pulse en un producto para ir al Boceto 4 (detalle).
  • “Añadir (+)” agrega una unidad al carrito.
  • El icono/botón “Carrito” del header lleva al Boceto 5.

Boceto 4 — Detalle de producto (Móvil)

Qué se ve: imagen(es), nombre, descripción, precio base, IVA y precio final.

Navegación:

  • Enlace a “Carrito” para revisar el pedido: Boceto 5.

Boceto 10 — Camarero (Tablet apaisada)

Qué se ve: pedidos “Recibido” para confirmar pago, pedidos “Listo cocina” para preparar y pedidos “Terminado” para entregar.

Navegación:

  • Seleccionar pedido “Recibido” y confirmar pago: pasa a “En preparación”.
  • Seleccionar “Listo cocina” y pasar a “Terminado” tras preparar extras.
  • Seleccionar “Terminado” y confirmar entrega: pasa a “Entregado”.

Flujos paso a paso desde la página principal

Estos flujos resumen cómo se realizarían las funcionalidades principales desde Inicio.

Flujo A — Cliente: realizar un pedido completo (Funcionalidad 2)

  1. Inicio (Boceto 1) → Iniciar sesión / Registrarse (Boceto 2).
  2. Acceder a la Carta (Boceto 3) y elegir tipo de pedido (Local / Llevar).
  3. Seleccionar productos (listado o detalle: Boceto 4) y añadirlos al carrito.
  4. Ir al Carrito (Boceto 5) para ajustar cantidades y confirmar pedido (estado “Recibido”).
  5. Pagar (Boceto 6) con tarjeta (simulado) o al camarero (cambia a “En preparación”).
  6. Ver Confirmación (Boceto 7) con nº de pedido y estados.
  7. Consultar el estado del pedido en el Perfil (Boceto 8).

Flujo B — Camarero: cobrar y entregar pedidos (Funcionalidad 2)

  1. Login como camarero (Boceto 2) → Panel camarero (Boceto 10).
  2. Seleccionar un pedido “Recibido” y confirmar pago → pasa a “En preparación”.
  3. Cuando cocina avisa “Listo cocina”, revisar extras y pasar a “Terminado”.
  4. Seleccionar pedido “Terminado” y confirmar entrega → pasa a “Entregado”.

Flujo C — Cocinero: preparar un pedido (Funcionalidad 3)

  1. Login como cocinero (Boceto 2) → Panel cocina (Boceto 9).
  2. Elegir un pedido “En preparación” y “quedárselo” → pasa a “Cocinando”.
  3. Marcar cada producto como preparado y finalizar → pasa a “Listo cocina”.

Flujo D — Gerente: crear/editar productos (Funcionalidad 1)

  1. Login como gerente (Boceto 2) → Panel gerente (Boceto 11).
  2. Crear/editar categorías en el panel lateral.
  3. Crear/editar productos con selector de categoría, imágenes y cálculo del precio final.
  4. Retirar productos de la carta actualizando “ofertado”.
